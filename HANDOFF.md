## CRITICAL BEHAVIOR RULES (project-specific)

1. **DO NOT** run `uv tool upgrade hcom` or `uv tool install --reinstall hcom` until upstream PR #54 ships in v0.7.19+. Either would re-pull PyPI v0.7.18 and silently lose the `--yolo` patch.
   - If accidentally run: rebuild from this repo (`cargo build --release --locked`) and re-swap via the mv-rename pattern (see "BINARY SWAP PATTERN" below).
2. **Binary swap pattern** (Linux text-file-busy): NEVER `cp` over a running binary — `cp` reopens the destination and fails with "Text file busy" if running processes hold it. Use mv-rename instead:
   ```bash
   INSTALLED=~/.local/share/uv/tools/hcom/bin/hcom
   cp "$INSTALLED" "$INSTALLED.<oldversion>.bak"          # rollback path
   mv "$INSTALLED" "$INSTALLED.<oldversion>.replaced-running"   # rename out of the way
   cp target/release/hcom "$INSTALLED"                    # new binary in place
   ```
   Running processes keep their open inode (kernel preserves it via fd); new invocations get the new binary. `.replaced-running` is auto-reaped when last fd closes.
3. **Source vs running binary distinction**: editing files in `src/` does NOT affect the live `hcom` CLI — rebuild + swap required. Docs-only changes (`README.md`, `docs/`, `FYI.md`, `HANDOFF.md`, `CLAUDE.md`, `AGENTS.md`, `skills/`) need no build.
4. **Fork rebaseability**: keep customizations in clearly-scoped paths (`docs/`, `FYI.md`, `HANDOFF.md`, `AGENTS.md`, `CLAUDE.md`, `PROJECT_INDEX.json`, `skills/`). Source changes should be upstream-PR-shaped (clean single commits, no fork-only docs in same commit).
5. **hcom config + DB locations**: `~/.hcom/config.toml` (config), `~/.hcom/hcom.db` (SQLite). Never commit `~/.hcom/*` to this repo.
6. **PROJECT_INDEX.json is auto-generated** by `/index` — never edit by hand. Modified-in-place-by-timestamp diffs can be discarded with `git restore PROJECT_INDEX.json`.
7. **DeepWiki MCP boundary** (per `CLAUDE.md` + `AGENTS.md`): use hosted `mcp__deepwiki__` (`https://mcp.deepwiki.com/mcp`) for public GitHub repos. Reserve local `mcp__deepwiki_local__` for private/offline analysis.
